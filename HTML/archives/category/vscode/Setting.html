VSCode 設定

在 VSCode 中，可以透過設定檔來隱藏不必要的資料，讓檔案列表看起來更整潔。例如，以下設定可以隱藏 `.vscode-server`、`.dbus` 和 `.shell.pre-oh-my-zsh` 資料夾：

  "files.exclude": {
      ".vscode-server": true,
      "**/.dbus": true,
      "**/.shell.pre-oh-my-zsh": true,
  }

另外，對於 C++ 的環境設定，可以使用以下設定來執行程式碼：

  "code-runner.runInTerminal": true,
  "code-runner.clearPreviousOutput": true,
  "code-runner.fileDirectoryAsCwd": true,
  "code-runner.executorMap": {
      "cpp": "g++ $fileName -o $fileNameWithoutExt && $dir$fileNameWithoutExt && echo '\n' "
  }

C 語言學習筆記

字串比較的三種方法

在 C 語言中，比較兩個字串的方法之一是使用 `strcmp()` 函數。以下是範例程式碼：

  #include <stdio.h>
  #include <string.h>
  
  int main() {
      char str1[] = "Hello";
      char str2[] = "Hello";
  
      if (strcmp(str1, str2) == 0) {
          printf("equal\n");
      } else {
          printf("not equal\n");
      }
  
      return 0;
  }

來源：https://shengyu7697.github.io/cpp-string-compare/

C 語言中的 `const` 和 `free` 用法詳解

`const` 關鍵字用於定義常數，不能改變其值。以下是幾個範例：

  • char* const b：表示指標 `b` 本身不能改變，但指向的內容可以改變。
  • const char* a：表示指標 `a` 指向的內容不能改變，但指標本身可以改變。
  • const char* const c：表示指標 `c` 本身和其指向的內容都不能改變。

範例程式碼：

  const char* a = NULL;
  a = "ABCD"; // 編譯通過，執行不會報錯
  //*a = 'a'; // 編譯報錯，因為內容不能修改
  
  char* const b = NULL;
  *b = "abcde"; // 編譯通過，因為內容可以修改
  //b = "aab"; // 編譯報錯，因為指標變數是一個常量，不能被修改
  
  const char* const c = "abcd"; // 內容和指標變數都不能被修改

來源：https://www.itread01.com/content/1550661862.html

VSCode 環境變數設定

在 VSCode 中，可以透過設定檔來定義環境變數。有兩種方法可以將 library 路徑加入設定中：

  1. 將 library 放入工作區：這樣每個資料夾都會有自己的 `.vscode` 資料夾和設定檔。
    2. 開啟命令選擇區 Ctrl+Shift+P 可以找到C/C++ 編譯組板c_cpp_properties.json 
    {
        "configurations": [
            {
                "name": "Linux",
                "includePath": [
                    "${workspaceFolder}/**",
                    "env:${ROOTSYS}/include"
                ],
                "defines": [],
                "compilerPath": "/usr/bin/gcc",
                "cStandard": "gnu17",
                "cppStandard": "gnu++14",
                "intelliSenseMode": "linux-gcc-x64"
            }
        ],
        "version": 4
    }
